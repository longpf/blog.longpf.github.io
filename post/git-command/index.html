Git 常用命令简记

30 Dec 2016, 14:51
git

前:HEAD关键字是指当前分支的最新一次提交,就是版本库中该分支上的最新版本

git init

git init 初始化一个repo,创建一个.git文件夹

git clone

git clone url newname nenamew 可以不要

git log

git log --oneline 每条log显示一行

git log --oneline --graph 图形化

git log branchname 指定分支log

git log --oneline branch1 ^branch2 在分支1,不在分支2,^为排除

git log --author=[author name]指定作者

git log --grep=keywords筛选commit信息

git log --grep=keywords --author=authorname 是OR的关系,如果想为AND则添加–all-match

git reflog

对reflog进行管理的命令,reflog是git用来记录引用变化的一种机制,比如记录分支的变化或者是HEAD引用的变化 当git reflog 不指定引用的时候,默认列出HEAD的reflog git会将变化记录到HEAD对应的reflog文件中,其路径为.git/logs/HEAD,分支的reflog文件都放在.git/logs/refs目录下的子目录中.

git reflog 可以看到reset的commit

git add

git add file

git add . 递归添加工作区的修改到暂缓区

git commit

git commit -m "message"

git commit -amend -m "message"取消上一次提交为新的提交信息

git reset

git reset HEAD filename 将staged(暂缓去)中的撤销到工作区,filename 不加则撤销全部staged

git reset --hard id HEAD指向变了,工作区,暂缓去修改都没了

git reset --mixed id HEAD指向变了,工作区有修改,暂缓区没有

git reset --soft id 相当于git reset --mixed id执行了add,HEAD指向变了,工作区没修改,暂缓去有修改

不加参数默认为git reset --mixed id

git revert

git revert commitid 反转撤销提交,撤销一个提交,(临时新建)一个提交,该提交是commitid的parent commit,这时候会有confict,需要解决再 add,commit,…

git revert -u id 创建一个反向的提交,不回提交

git revert : 生成一个新的提交来撤销某次提交,push的时候不会有影响,HEAD指向新生成的commit

git reset : 回到历史的某一次提交,这次提交之后的内容会被删除,push的时候可能会有冲突,HEAD指向历史的某次commit

回滚操作

先 git reset --hard commitid 到要回滚的commitid,再 git push -f 强制推送到远程,这个-f的话就不必pull了,如果不带-f会提示远程有更新,提示你pull.这个强制推送会 覆盖远程的代码

git rm

git rm

git stash

git stash 将工作区,暂缓区的内容压栈,即货到上一次最新的提交

在哪个分支入栈,则在哪个分支找回

git stash list 显示栈的list

git stash pop 出栈

git stash apply ... 取出栈内指定的快照(list不会删除)

git stash drop ... 删除指定的快照

git stash clear 删除整个list

git branch

git push -u orgin branchname将一个分支推到远程

git branch 列出本地的分支

git branch -a 列出远程和本地的分支

git branch -r 列出远程的分支

git branch -d branchname 删除本地分支

git push origin --delete branchname 删除远程分支

git branch -r -d origin/branchname 删除远程分支

git checkout

git checkout -b branchname 创建并切换到该分支

git checkout -b localbranchname origin/branchname将远程分支拉到本地

git checkout filename删除工作区某个文件的修改

git merge

git merge branchname 合并分支到当前分支

git tag

git tag -a 6.6.3 -m "6.6.3版本" commitid -a会允许你添加一些信息

git show tagname

git tag -d

git push origin :refs/tags/tagname 删除远程的tag

‘git push origin tagname’ 将tag推到远程

git push origin --tags 将tag推到远程

git fetch

将远程新的分支和数来下载下来

git fetch 获取当前仓库

git fetch reponame

git fetch --all取到远程全部的repo

git pull

git pull = git fetch + git merge 把取下来的分支的head merge到当前的分支,产生一个新的commit

git pull --rebase 这个会让 tree是一条直线,不回出现并行的tree,方便 reset和revert –rebae ,将会执行 git rebase来取代原来的git merge

git rebase

–rebase 不会产生合并的提交,它将会将本地的所有提交临时保存为补丁(patch),放在’.git/rebase’目录中,然后将当前分支更新到最新的分支尖端,最后把保存的不定应用到分支上.

rebase过程中,可能会出现冲突,git会停止rebse让你解决冲突,解决后在git add去更新这些内容,然后无需执行commit,只需要:

git rebase --continue就会继续打余下的补丁

git rebase --abort将会终止rebase,当前分支将会回到rebase之前的状态

git push

git push origin branchname将当前分支推到远程origin的某个分支
